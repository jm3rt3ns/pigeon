# Domain Specific Language for defining tables using markdown and producing rich table functionality in a more concise way

The following definition:
* [Example Definition](./src/table_definition.md)

produces:
![](./image.png)

The common problem with setting up tables is that it can require touching a large number of files:
* column definitions
* service calls (axios)
* rendering
* setup of sorting/filtering/searching
* handling of mutation

While this can be useful, this is an experiment to see if the work can be "conventionalized" in a way that simplifies the developer process for adding new tabular data, while still providing a high degree of flexibility to handle different tables
